Программа подготовки файлов для проведения оценки качества работы сотрудников 2-й линии технической поддержки (Отдел информационно-технологического сопровождения центральной бухгалтерии).

Группа - ЦА 1С_Группа сопровождения (ПУНФА, ПУиО, ПУК).


I. Порядок работы с программой:

    1. Подготовка файла для оценки:

        а. Выберите вкладку "Подготовка файла";

        б. Нажмите кнопку "Загрузить файл с данными" и выберите файл, содержащий выгрузку обращений из СУЭ.

        в. Программа обработает файл и отобразит итоги своей работы в таблице ниже. Нажав кнопку "Сохранить в формате excel", таблицу можно сохранить в файл excel;

        г. При помощи выпадающего списка можно выбрать один из проверяющих регионов, при этом в таблице будут отражены соответствующие записи по выбранному региону.
Если необходимо отобразить все записи из таблицы, то следует выбрать пункт "Все регионы"

        д. При необходимости, можно изменить количество задач, которое будет отбираться по каждому пользователю. Для этого необходимо ввести количество (цифра от 1 до 10) в специальное поле и нажать кнопку "Установить".

        е. В случае несоответствия файла необходимым требованиям, программа выведет на экран сообщение об ошибке.


    2. Порядок расшифровки файла с оценками:

        а. Выберите вкладку "Расшифровка файла";

        б. Нажмите кнопку "Загрузить файл с данными" и выберите файл для расшифровки;

        в. Программа обработает файл и отобразит итоги своей работы в таблице ниже. Нажав кнопку "Сохранить в формате excel", таблицу можно сохранить в файл excel;

        г. При необходимости более детально посмотреть выставленные оценки по задачам, а также комментарии оценивающего, необходимо нажать кнопку "Детализация".
        В открывшемся окне, в выпадающем списке необходимо выбрать сотрудника. Программа отобразит в таблице оцененные задачи по выбранному сотруднику.
        Данные можно сохранить в файл excel.

        д. В случае несоответствия файла необходимым требованиям, программа выведет на экран сообщение об ошибке.


II. Требования к формату файлов, обрабатываемых программой.

    1. Формат файла для шифрования.

        а. Файл должен быть выгружен из СУЭ ФК;

        б. Файл в обязательном порядке должен содержать следующие столбцы:
            - Дата/время регистрации;
            - Номер;
            - Технический статус;
            - Описание;
            - Ответственный (сотрудник);
            - Плановое время выполнения;
            - Просрочен крайний срок;
            - Фактическое время выполнения;
            - Код закрытия;
            - Описание решения;
            - Кем решен (группа);
            - Кем решен (сотрудник);
            - Количество уточнений;
            - Количество возобновлений;
            - Аналитические признаки;
            - Приоритет;
            - Статус;
            - Причина ожидания;
            - Кем решен.

        в. Файл должен содержать только выполненные (закрытые) обращения, обязательно должна быть указана фактическая дата выполнения работ;

        г. Желательно, чтобы в файле в столбце "Кем решен (сотрудник)" были только сотрудники ОИТ СЦБ из списка (список сотрудников приведен на вкладке "список сотрудников");

    2. Формат файла для расшифровки.

        а. Файл с оценками в обязательном порядке должен содержать столбец № 1, название которого содержит уникальный идентификатор ключа

        б. Столбец с названием ключа обязательно должен быть первым.

        в. Последовательность строк в файле значения не имеет.


III. Алгоритм работы программы по подготовке файла:

    1. Загружается список сотрудников (список приведен на вкладке "Сотрудники ОИТ СЦБ");

    2. Каждому сотруднику в случайном порядке присваивается определенный цифровой код. (Для расшифровки, информация сохраняется в специальный ключевой файл.);

    3. Определяется количество обращений, решенных каждым сотрудником, и случайным образом выбирается определенное количество обращений (по умолчанию - 5, это значение можно изменить, введя требуемое количество в поле ввода и нажав кнопку "Установить"), которые будут оцениваться;

    4. Определяется регион оцениваемого сотрудника, и случайным образом выбирается регион, в которой будет отправлено данное обращение на оценку;

    5. Время работы по заявки рассчитывается в часах как разница между временем регистрации обращения и временем фактического решения (не уверен, что это правильно);

    6. Для каждой заявки определяется уровень сложности (если оператор поставил уровень сложности при исполнении заявки).
    Также рассчитывается средний уровень сложности для каждого сотрудника, и выводится справочно в итоговой таблице с оценками;

    7. Добавлена колонка "Количество уточнений", как мне кажется также важный показатель для оценки. При необходимости можно легко удалить данную колонку;

    8. Последняя колонка показывает возможное (реализован простой поиск, по ключевым словам, по тексту обращения) наличие вложенных файлов в заявке:
        - Да - в обращении скорее всего имеются вложения, которые позволяют более точно понять проблему;
        - Нет - скорее всего дополнительных вложенных файлов нет.


IV. Алгоритм работы программы по дешифровки файла

    1. Загружается файл который необходимо расшифровать. При этом в обязательном порядке в файле должны быть заполнены три столбца с оценками (Решение, Время решения, Кол-во возвратов). Оценки представляют из себя целые числа от 0 до 4, и должны быть проставлены в каждой ячейке. Другие символы и пропуски не допускаются;

    2. На основании информации из первого столбца производится поиск ключевого файла для расшифровки (ключевые файлы располагаются в специальной папке);

    3. Если ключевой файл найден, то производится сопоставление уникального идентификатора сотрудника из загруженного файла с информацией из ключевого файла;

    4. Расшифрованный файл выводится в таблице на экран. Нажав кнопку "Сохранить в формате excel", таблицу можно сохранить в файл excel;

    5. Нажав кнопку "Детализация" откроется окно, в котором можно по конкретному сотруднику посмотреть детальную информацию, по оценке обращений.

    6. Если ключевой файл не найден или загруженный файл не соответствует формату, программа отобразит на экране сообщение об ошибке.
